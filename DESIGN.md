## Overview

Gallery Explorer should feel calm, quiet, and highly legible. It uses Material 3 components and must adapt to the user's
device theme setting, presenting a coherent light or dark scheme without changing the product's overall character. The
chrome should recede so storage destinations, thumbnails, and full-screen media do the talking. The visual tone is not
cinematic gloss or neon futurism. It is restrained, practical, and polished, with just enough softness to feel modern.

The accent color is a cool periwinkle blue used with discipline. It highlights storage icons, folder glyphs, sorting
controls, video affordances, and the active playback track, but it does not wash across entire screens. The surrounding
UI should remain tonal and balanced in both light and dark modes.

The overall impression should be that of a focused media utility that happens to be elegant, not an ornamental showcase.
When in doubt, remove decoration and let scale, contrast, and spacing establish hierarchy.

## Colors

The core palette is a pair of Material 3 tonal schemes: one for light mode and one for dark mode. Components should use
the scheme that matches the active device setting. The screenshots in this repo show dark-mode examples, but those
screenshots are illustrative, not normative for the product's default mode.

- `light-*` tokens define the light Material 3 scheme built from soft off-whites, cool grays, and a deeper blue primary.
- `dark-*` tokens define the dark Material 3 scheme built from charcoal surfaces, pale text, and a lighter blue primary.
- The `primary` role is the only clearly expressive accent in either scheme. It should remain cool, slightly
  desaturated, and sparingly used.
- `surface-variant` style roles are the workhorse neutrals for folder tiles, empty thumbnail states, and inactive
  utility surfaces.
- `on-surface` and `on-surface-variant` style roles carry hierarchy. Major text should stay high contrast, while
  unsupported files, timer text, and secondary metadata should be quieter.

## Typography

Typography uses `Google Sans` throughout. There is no decorative display face and no serif contrast. The system relies
on the font's rounded geometry and clean apertures to keep the UI friendly while staying functional.

- Large screen titles and breadcrumbs should feel open and breathable rather than dense or technical.
- Tile labels and metadata should stay compact, centered, and readable at a glance.
- Semi-bold emphasis is used selectively for titles and key row labels. Most copy remains regular weight.
- The system should avoid mixing many weights on the same screen. The product reads best when hierarchy comes from size
  first, then weight.

## Layout

Spacing follows a strict 8px rhythm with a few purposeful 20px and 24px expansions for containers and immersive
controls.

- Home uses a single-column list with generous horizontal gutters and oversized storage cards.
- Explorer uses a rigid 3-column square grid with 16px outer padding and 8px internal gaps.
- File captions sit directly under their tiles with a tight 8px separation and remain single-line whenever possible.
- Viewer screens are full-bleed and vertically paged. The content owns the screen; chrome is layered on top rather than
  boxed around it.
- Long titles should scroll horizontally instead of wrapping. This keeps the interface clean and preserves the strong
  horizontal line at the top of the screen.

## Elevation & Depth

Depth is achieved almost entirely through tonal separation, not cast shadows. The interface should feel flat,
deliberate, and slightly dense.

- Cards and controls rise by stepping to the next tonal container in the active Material 3 scheme, not by floating
  dramatically.
- Shadows should be absent or imperceptible.
- The only notable sense of depth comes from media itself, the tonal contrast between layers, and the top and bottom
  viewer scrims that help controls sit above imagery or video.
- Small fallback surfaces may use a subtle tonal elevation equivalent to 2dp, but the result should still read as flat.

## Shapes

The shape language is one of the clearest parts of the product identity.

- Primary containers use a large 24px radius.
- Square file tiles should feel chunky and friendly, never sharp or severe.
- Circular affordances are reserved for playback and compact media actions.
- Avoid mixing multiple unrelated corner radii on the same screen. Most surfaces should resolve to either the large 24px
  tile radius or a full circle.

## Components

The home storage card is a wide, low, softly rounded slab with a blue storage icon on the left and a single strong
label. It should read like a destination, not like a settings row.

Explorer tiles are square and image-led. Directories use a solid neutral tile with a centered blue folder glyph. Images
and videos use edge-to-edge thumbnails with the same large radius so the grid feels uniform. Unsupported files use the
same tile geometry but lower-contrast icon and label treatment.

The explorer top bar uses a large breadcrumb title with bullet separators, plus compact sort controls on the right.
Those controls should stay visually lightweight and should borrow the accent color without overpowering the title.

Viewer chrome is intentionally sparse. The image viewer keeps only a minimal top bar. The video viewer adds gradient
scrims at the top and bottom, an active slider track keyed to the current theme's primary color, compact elapsed and
remaining times, and three circular transport buttons centered below the scrubber.

Use the `*-light` component tokens when the app is in light mode and the `*-dark` component tokens when the app is in
dark mode. Do not force dark styling when the device is set to light mode.

Empty states should remain simple: one icon, one headline, one or two lines of supporting copy, and a single clear call
to action.

## Do's and Don'ts

- Do use Material 3 components that adapt cleanly to the user's light or dark device setting.
- Do keep the app tonal and uncluttered.
- Do use the blue accent sparingly for action and recognition.
- Do preserve the large 24px tile radius across cards, folders, thumbnails, and placeholders.
- Do let media fill the frame in viewer screens.
- Don't force dark surfaces or dark component variants when the device is set to light mode.
- Don't introduce glossy gradients, glassmorphism, or visible drop shadows.
- Don't turn the accent blue into a full-screen background treatment.
- Don't crowd the layout with dense metadata, dividers, or secondary actions.
- Don't mix sharp rectangles with the established rounded tile system.
